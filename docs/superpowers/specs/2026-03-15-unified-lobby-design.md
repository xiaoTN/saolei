# 统一入口界面设计规范

日期：2026-03-15
状态：已审批

## 目标

去掉当前「选择单人/双人模式」的入口屏（`modeScreen`），打开游戏直接进入统一大厅页面，用户在同一个页面完成单人开始和联机操作。

## 最终方案（方案 B 变体）

### 页面结构（单栏，手机优先）

```
💣 多边形扫雷
─────────────────────────
[ 棋盘形状 ]  （7个形状按钮，与现有单人面板相同）
[ 难度选择 ]  （简单 / 中等 / 困难 / 地狱 / 自定义）
[ 自定义滑块 ]（行数 / 列数 / 雷数，custom 时展开）
[ 格子数 / 雷率预览 ]
─────────────────────────
[ ▶ 开始游戏 ]  [ 创建房间 ]
  （蓝色，主）    （紫色，次，同宽或略窄）
─────────────────────────
— 或加入已有房间 —
[ 输入房间码 ][ 加入 ]
[ 当前可加入的房间列表 ]
```

### 关键设计决策

1. **设置面板共用**：棋盘形状、难度、自定义滑块、格子数/雷率预览，单人和联机创建房间**共用同一套设置**，不做区分。
2. **「创建房间」复用当前设置**：点「创建房间」时，直接使用面板上已选的 sides / difficulty / rows / cols / mines 参数，无需单独的联机设置面板。
3. **「加入房间」区域独立**：分隔线下方为加入区域，包含手动输入房间码和房间列表，与设置无关。
4. **移除 modeScreen**：删除现有的「单人游戏 / 双人联机」模式选择屏，页面加载后直接显示统一大厅。
5. **「返回」按钮调整**：游戏进行中的「← 返回」回到统一大厅（原来的 `startScreen` / `lobbyScreen` 逻辑合并）。

## 涉及文件

| 文件 | 改动内容 |
|------|---------|
| `index.html` | 删除 `modeScreen`，合并 `startScreen` + 创建房间部分为新的统一大厅区块；联机加入区域保留在下方 |
| `style.css` | 新增「双按钮行」样式（`.action-row`），删除 `.mode-screen` / `.mode-card` 相关样式 |
| `game.js` | 删除 `enterSinglePlayer()` / `enterMultiplayer()` / `backToModeSelect()`；`startGame()` 和 `createRoom()` 直接读取统一面板的设置；`backToMenu()` 回到统一大厅 |

## 不改变的部分

- 等待室屏（`waitingScreen`）逻辑不变
- 游戏界面（`gameScreen`）逻辑不变
- `multiplayer.js` 的 WebSocket 通信逻辑不变
- 棋盘形状按钮渲染函数 `renderShapeButtons` 不变（复用）
