# 统一入口界面设计规范

日期：2026-03-15
状态：已审批

## 目标

去掉当前「选择单人/双人模式」的入口屏（`modeScreen`），打开游戏直接进入统一大厅页面，用户在同一个页面完成单人开始和联机操作。

## 最终方案（方案 B 变体）

### 页面结构（单栏，手机优先）

```
💣 多边形扫雷
─────────────────────────
[ 棋盘形状 ]  （7个形状按钮，与现有单人面板相同）
[ 难度选择 ]  （简单 / 中等 / 困难 / 地狱 / 自定义）
[ 自定义滑块 ]（行数 / 列数 / 雷数，custom 时展开）
[ 格子数 / 雷率预览 ]
─────────────────────────
[ ▶ 开始游戏 ]  [ 创建房间 ]
  （蓝色，主）    （紫色，次，同宽或略窄）
─────────────────────────
— 或加入已有房间 —
[ 输入房间码 ][ 加入 ]
[ 当前可加入的房间列表 ]
```

### 关键设计决策

1. **原单人设置面板升级为共用设置面板**：棋盘形状、难度、自定义滑块、格子数/雷率预览，「开始游戏」和「创建房间」均读取此面板参数，不做区分。
2. **「加入房间」区域独立**：分隔线下方为加入区域，包含手动输入房间码和房间列表，与设置无关。房间列表行为与现有 `lobbyScreen` 一致：进入大厅时自动拉取一次，并保留「刷新」按钮，无房间时显示「暂无等待中的房间」。
3. **移除 modeScreen**：删除现有的「单人游戏 / 双人联机」模式选择屏，页面加载后直接显示统一大厅。
4. **「返回」按钮调整**：游戏进行中「← 返回」回到统一大厅，设置面板**保留上一局的参数**（现有行为不变）。联机游戏返回时断线通知逻辑由现有 `backToMenu()` 内已有的 `MP.leaveRoom()` 处理，无需新增逻辑。
5. **自定义参数校验**：复用现有单人游戏的校验逻辑，`createRoom()` 读取参数前不做额外校验（与现在一致）。
6. **函数删除前置检查**：实现时先全文搜索 `enterSinglePlayer` / `enterMultiplayer` / `backToModeSelect` 的所有引用，确认清除干净后再删除。

## 涉及文件

| 文件 | 改动内容 |
|------|---------|
| `index.html` | 删除 `modeScreen`，合并 `startScreen` + 创建房间部分为新的统一大厅区块；联机加入区域保留在下方 |
| `style.css` | 新增「双按钮行」样式（`.action-row`），删除 `.mode-screen` / `.mode-card` 相关样式 |
| `game.js` | 删除 `enterSinglePlayer()` / `enterMultiplayer()` / `backToModeSelect()`；`startGame()` 和 `createRoom()` 直接读取统一面板的设置；`backToMenu()` 回到统一大厅 |

## 不改变的部分

- 等待室屏（`waitingScreen`）逻辑不变
- 游戏界面（`gameScreen`）逻辑不变
- `multiplayer.js` 的 WebSocket 通信逻辑不变
- 棋盘形状按钮渲染函数 `renderShapeButtons` 不变（复用）
