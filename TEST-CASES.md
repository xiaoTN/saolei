# 测试用例

本文档记录多边形扫雷游戏的手工测试用例，每次发布前应按此清单验证。

---

## 1. 基础界面测试

### 1.1 开始界面
- [ ] 页面正常加载，显示标题"多边形扫雷"
- [ ] 四个多边形模式按钮（三角形/正方形/六边形/八边形）可点击
- [ ] 五个难度按钮（简单/中等/困难/地狱/自定义）可点击
- [ ] 预览信息（格子数、雷密度）随选择变化
- [ ] "开始游戏"按钮可点击

### 1.2 游戏界面
- [ ] 点击"开始游戏"后正确切换到游戏界面
- [ ] 顶部状态栏显示：返回按钮、旗子数、计时器、重置按钮
- [ ] 棋盘正确渲染（根据选择的多边形模式）
- [ ] 点击"返回"按钮可返回开始界面

---

## 2. 设置锁定/解锁测试 ⚠️ 重要

### 2.1 返回菜单后设置解锁
**测试步骤**:
1. 点击"开始游戏"进入游戏
2. 在棋盘上进行首次点击（开格）
3. 点击"← 返回"按钮返回菜单
4. 尝试点击"三角形/六边形/八边形"按钮

**预期结果**: 多边形模式按钮可点击，预览信息正确更新

**关联修复**: `game.js` - `showStartScreen()` 中添加 `_setSettingsLocked(false)`

---

## 3. 游戏操作测试

### 3.1 左键点击 - 标旗
- [ ] 左键点击未揭示格子，显示旗子图标
- [ ] 再次左键点击已标旗格子，取消标旗
- [ ] 旗子计数正确更新

### 3.2 右键/长按 - 开格
- [ ] 右键点击未揭示格子，揭示格子
- [ ] 长按（450ms）未揭示格子，揭示格子
- [ ] 首次点击安全（不会踩雷）

### 3.3 数字格快速开雷（Chord）
- [ ] 点击已揭示的数字格
- [ ] 若周围旗子数等于数字，自动揭示周围未标旗格子

### 3.4 胜利/失败
- [ ] 揭示所有非雷格子后显示胜利弹窗
- [ ] 点击雷后显示失败弹窗
- [ ] 游戏结束后格子 hover 不变色
